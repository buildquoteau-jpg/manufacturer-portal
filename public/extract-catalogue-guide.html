How to Import a New Manufacturer Catalogue

Using the /extract-catalogue Claude Code skill to turn a PDF into database-ready product data

What this does

The /extract-catalogue skill is a reusable workflow built into Claude Code. You give it a manufacturer's PDF catalogue, and it reads through the document and produces a SQL file ready to paste into Supabase — complete with product profiles, colour options, fixings, and component relationships.

The whole process usually takes 5–15 minutes depending on catalogue size.

Before you start — have these ready:

  • The manufacturer's PDF catalogue saved to your computer (e.g. Downloads/ntw-2025.pdf)
  • The manufacturer's website URL
  • Access to the Supabase SQL Editor for the BuildQuote project
  • Claude Code open on your computer

Step-by-step instructions

Open Claude Code and navigate to the manufacturer portal project

Open Claude Code (the terminal application). At the bottom of the screen you'll see a file path — make sure it shows the manufacturer portal folder:

C:\Users\melia\OneDrive\Desktop\manufacturer-portal

If it shows a different folder, type cd followed by the correct path and press Enter.

Start a fresh chat

Press Ctrl + R (or click New Chat) to start a clean conversation. This stops the old context from interfering with the new extraction.

Note the full path to your PDF

You need to give Claude the exact file path. The easiest way:

  • Windows: Hold Shift and right-click the PDF → Copy as path
  • Mac: Right-click the PDF → hold Option → Copy as Pathname

The path will look something like:

C:\Users\melia\Downloads\manufacturer-catalogue-2025.pdf
Run the skill

In the Claude Code chat, type the following command and press Enter:

/extract-catalogue C:\Users\melia\Downloads\manufacturer-catalogue-2025.pdf

Replace the file path with your actual PDF path from Step 3.

Tip: You can drag the PDF file directly into the Claude Code chat window — it will insert the path for you automatically.
Answer Claude's questions

Claude will pause and ask you a few things before it starts extracting. Common questions include:

Claude asksWhat to answer
Does this manufacturer already exist in the database?Yes or No. If Yes, log into Supabase → Table Editor → manufacturers → copy the UUID of that row and paste it in.
This product doesn't fit a standard category — which should it be?Choose: Decking, Cladding, or Screening & Fencing
This component has no SKU — should I generate one or leave it blank?Usually say "generate one" — Claude will create a sensible code like MFG-CLIP-01
Review the extracted data

Claude will present a summary of everything it found before producing the SQL. Check:

  • Product names and codes look correct
  • Each product is in the right category (Decking / Cladding / Screening & Fencing)
  • Dimensions are in mm (e.g. "138mm x 29mm"), not inches or feet
  • Lengths are in metres (e.g. 4.88m), not feet
  • Colours look complete — no obvious ones missing
  • Components and fixings are all listed

If anything looks wrong, tell Claude — e.g. "The colour 'Sandstone' is missing from the Avenue Range" — and it will fix it before generating the SQL.

Copy the SQL output

Once you're happy with the data, Claude will output a SQL script. It will look like this:

-- ============================================================
-- NewTech Wood — Catalogue Import
-- ============================================================

INSERT INTO manufacturers ...
INSERT INTO systems ...
INSERT INTO system_colours ...
INSERT INTO components ...
INSERT INTO system_components ...

Select all of the SQL and copy it (Ctrl + A then Ctrl + C, or click the copy button if visible).

Run the SQL in Supabase

Log in to supabase.com → open the BuildQuote project → click SQL Editor in the left sidebar.

Click New query, paste your SQL, and click Run (or press Ctrl + Enter).

Success — you'll see a green confirmation and row counts for each table that was populated.
If you get an error — copy the error message, go back to Claude Code, and paste it in. Claude will diagnose and fix the SQL for you.
Check the live portal

Go to mfp.buildquote.com.au and log into the admin area. The new manufacturer and their products should now appear in the Add new supplier wizard under Step 2 (Choose manufacturer).

You can also go to Admin → suppliers and create a test widget to verify everything loaded correctly.

Tips & common issues

The PDF is scanned / image-only

If the PDF is a scanned document (no selectable text), Claude will still try to read it using visual analysis — but accuracy is lower. For best results, ask the manufacturer for a digital PDF, or use Adobe Acrobat's OCR feature first to make the text selectable.

The catalogue is very large (100+ pages)

Tell Claude upfront: "This is a 120-page catalogue — focus on pages 10–80 which contain the product profiles." You can always run the skill again for accessories/components on a second pass.

Colours are only shown as swatches with no names

Take a photo of the swatch page and mention the visible names to Claude, or check the manufacturer's website colour selector. Claude will flag this and ask.

A manufacturer has multiple catalogues (decking + cladding separately)

Run the skill once per PDF. On the second run, when Claude asks if the manufacturer already exists, say Yes and provide the UUID — it will add to the existing data rather than duplicating the manufacturer row.

Products are in the wrong category after import

You can fix this directly in Supabase: Table Editor → systems → find the row → edit the category field. Only three values are valid: Decking, Cladding, Screening & Fencing (exact spelling including the ampersand).

Product images and colour swatch photos are not imported from the PDF — these are added separately by uploading images and pasting the URL into the hero_image_url or image_url fields in the Supabase Table Editor.
